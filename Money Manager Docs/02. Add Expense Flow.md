## üß© Part 1: Entry Points

Users can add expenses from:

1. **Home Screen FAB** ‚Üí "+" button (floating action button)
2. **Quick Add Widget** ‚Üí (Future: Today's expense shortcut)
3. **Recurring Bills** ‚Üí Create from template

---

## üß© Part 2: Form Structure

### Screen 1: Expense Details

**Fields** (in order):

1. **Amount** (required, numeric input)
   - Currency symbol (‚Çπ for INR by default)
   - Decimal support
   - Quick presets: ‚Çπ100, ‚Çπ500, ‚Çπ1000 (optional)

2. **Category** (required, dropdown/picker)
   - Shows predefined categories with icons
   - "Add custom category" option at bottom
   - Selected category shows color indicator

3. **Date** (auto-filled with today)
   - Tap to open date picker
   - Show: "Today", "Yesterday", "Select date"

4. **Time** (auto-filled with current time)
   - Tap to open time picker (HH:MM format)
   - Only for single expenses (not recurring)
   - Optional: "Not specified" option

5. **Description** (optional, text field)
   - Placeholder: "Add details (lunch, groceries, etc.)"
   - Max 100 characters

6. **Notes** (optional, text field)
   - Placeholder: "Any additional notes"
   - Max 200 characters

---

### Screen 2: Recurring Setup (Conditional)

Only shown if "Make this recurring?" toggle is ON.

**Toggle**: "Make this recurring?"

If YES, show:

1. **Frequency** (required)
   - Radio buttons: Daily / Weekly / Monthly
   - Default: Monthly

2. **Start Date** (required)
   - Defaults to today
   - Cannot be in the future

3. **End Date** (optional)
   - "Never ends" (default)
   - "Ends on" ‚Üí Date picker

4. **Reminder** (optional)
   - Days before reminder: 0, 1, 3, 7
   - Default: 0 (on the day)

## üß© Part 3: Validation Rules

| Field | Rules | Error Message |
|-------|-------|---|
| Amount | > 0, decimal OK | "Amount must be greater than 0" |
| Category | Must select | "Please select a category" |
| Date | Not in future | "Date cannot be in the future" |
| Description | Max 100 chars | Auto-trim, no error |
| Recurring Start | Not in future | "Start date must be today or earlier" |

## üß© Part 5: Actions

### Save Button
- Bottom right / Primary action
- Disabled until: amount + category filled
- On save:
  - Create Expense in Swift Data
  - If recurring: Create RecurringExpense + generate first instance
  - Show toast: "Expense added" / "Recurring expense set"
  - Dismiss to Home Screen

### Cancel Button
- Top left / Back action
- Warn if data entered: "Discard changes?"
- Dismiss to previous screen

### Delete (Edit Mode Only)
- Long-press or swipe action
- Confirm deletion
- Remove from Swift Data
